# pipelines

## Purpose

Defines the operational workflow for deploying, reviewing, promoting, rolling back, and monitoring infrastructure changes.

## Contains

- deploy order and promotion gate guidance
- change-set review, rollback, and drift-detection models
- Service Catalog packaging and versioning workflow docs

## How It Is Used

Use these documents as the human-readable deployment policy behind the scripts in `scripts/` and the stack manifests in `stacks/`.

## Notes

This directory describes process and guardrails, not executable automation.
